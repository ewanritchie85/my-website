Project repository

For my final project of Northcoders' data engineering bootcamp my small team were tasked with creating an ETL pipeline that periodically extracted new data from a Postgres OLTP database for storage in an AWS S3 'ingestion' bucket.
Upon data arrival in the 'ingestion' bucket, AWS EventBridge triggered the transform lambda function which processed the data and stored it in a second 'processed' bucket.
The data was then loaded into a Postgres OLAP database using a third lambda function which was triggered every twenty minutes to load processed data into an OLAP databse where clean data was analysed using Tableau.
Each triggering of the lambda functions was logged with Cloudwatch and significant errors were sent to the team via email.
The pipeline was deployed using Terraform IaC and the code was tested using Pytest during deployment with Github Actions. All login credentials were kept private using AWS Secrets Manager
